Sideways Information Passing

Implementation details

Merge Joins and Index Scans:

These work on the B+ indexes. Checking if a triple should be skipped too often results in a major CPU overhead, and in a significant drop in L2 hit rate

The system only checks for skips when a new page is about to be read into memory

The CPU overhead is once per page, which is amortized to nearly zero

The time saved by not reading whole pages is significant
